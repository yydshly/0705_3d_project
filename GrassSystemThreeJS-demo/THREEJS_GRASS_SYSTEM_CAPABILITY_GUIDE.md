# Three.js Grass System 能力沉淀说明

这份文档记录我们基于 `GrassSystemThreeJS` 做过的理解、改造和发布流程。它不是原库 API 文档，而是面向后续复用的实践说明：这个库能体现哪些能力、如何操作、以及这些能力可以服务哪些真实场景。

## 1. 我们可以沉淀的技能

### 1.1 Web 3D 技术理解

这个项目帮助我们建立了一条清晰的技术链路：

```text
业务代码 / Three.js
  -> WebGL
  -> Shader
  -> GPU
  -> 浏览器 canvas 画面
```

可以沉淀的知识点：

- Three.js 是 WebGL 上层封装，负责场景、相机、模型、材质、灯光、动画和加载器。
- WebGL 是浏览器访问 GPU 的底层图形接口。
- Shader 是运行在 GPU 上的小程序，决定顶点如何移动、像素如何着色。
- GPU 适合大规模并行计算，适合草叶、粒子、风场、后期效果这类高密度视觉任务。

### 1.2 实时 3D 场景构建能力

这个 demo 不只是“显示草”，而是一个完整实时场景：

- 地面：PBR 土壤贴图、位移、湿度、沟壑、根系痕迹。
- 植被：GPU 实例化草叶、密度控制、高度控制、覆盖范围控制。
- 风场：用噪声和时间参数驱动草叶摆动。
- 苔藓：通过 shader 和局部遮罩模拟潮湿阴影区域生长。
- 模型：加载车辆 GLB，并与地面、草、苔藓产生视觉关系。
- 镜头：远景、近景、旋转、推拉、呼吸感和电影化节奏。
- 后期：色调、景深、雾、云、光影和发布画幅。

### 1.3 参数化演示设计

这个项目适合沉淀“参数驱动演示”的方法：

- 不把效果写死，而是用参数控制密度、高度、风速、风向、苔藓覆盖、土壤湿度等。
- 用时间轴驱动参数变化，形成“从无到有”的构建过程。
- 用导演脚本组织镜头，让技术能力变成观众能理解的故事。

这种方法可以迁移到很多 3D 展示项目：产品展示、建筑展示、数据可视化、交互官网、教学动画。

### 1.4 3D 内容发布流程

我们还沉淀了一条从网页演示到发布视频的流程：

```text
Three.js 实时演示
  -> 浏览器录制 WebM
  -> MiniMax 生成旁白和音乐
  -> ffmpeg 转 MP4
  -> 生成封面、字幕、发布文案
  -> 平台发布
```

对应能力：

- 浏览器 canvas 录制。
- 音视频对齐。
- MiniMax TTS 旁白生成。
- MiniMax Music 背景音乐生成。
- WebM 转 MP4。
- 抽帧生成封面。
- 生成 SRT 字幕和发布说明。

## 2. 当前项目操作步骤

### 2.1 启动演示

安装依赖后启动本地服务：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

页面中主要入口：

- `Play Full Scene Demo`：播放完整场景演示。
- `Record System Film`：录制用于发布的视频版演示。
- `Film Captions`：控制录制字幕叠加。
- `Film Music`：控制背景音乐。
- `Film Voiceover`：控制旁白。

### 2.2 重新生成旁白和音乐

生成 MiniMax 旁白和背景音乐：

```bash
npm run audio:minimax
```

只重新生成旁白：

```bash
npm run audio:minimax -- voiceover
```

只重新生成音乐：

```bash
npm run audio:minimax -- music
```

默认输出：

- `public/system-film-voiceover.mp3`
- `public/system-film-music.mp3`

当前默认旁白音色：

```text
male-qn-jingying
```

可以通过环境变量覆盖：

```bash
SYSTEM_FILM_TTS_VOICE_ID=male-qn-jingying
SYSTEM_FILM_TTS_SPEED=0.8
SYSTEM_FILM_TTS_PITCH=-2
```

### 2.3 录制 WebM

在网页中点击：

```text
Record System Film
```

浏览器会下载类似文件：

```text
grasssystem-system-build-film-*.webm
```

默认下载位置通常是：

```text
C:\Users\<用户名>\Downloads
```

### 2.4 生成发布包

将最新 WebM 转成平台更通用的 MP4，并生成封面、字幕和文案：

```bash
npm run publish:film
```

输出目录：

```text
publish/system-film-<timestamp>/
```

输出内容：

- `system-film-16x9.mp4`：发布视频。
- `cover-title.png`：封面。
- `cover-raw.png`：备用封面。
- `system-film-captions.srt`：字幕。
- `publish-notes.md`：标题、简介、标签。
- `source.webm`：原始录制备份。

当前发布规格：

```text
MP4 / H.264
1920x1080
16:9
60fps
AAC 48k / 192k
```

发布脚本会优先使用：

- `public/system-film-voiceover.mp3`
- `public/system-film-music.mp3`

重新混音，而不是保留 WebM 里的旧音轨。

## 3. 这个库体现的核心能力

### 3.1 大量草叶的实时渲染

核心价值是通过 GPU 实例化渲染大量草叶，而不是在 CPU 上逐根绘制。这样可以在浏览器中显示成千上万根草，同时保持实时帧率。

适合复用到：

- 草地
- 毛发
- 森林
- 粒子群
- 人群
- 大量重复产品陈列

### 3.2 Shader 驱动视觉变化

草叶摆动、苔藓覆盖、土壤凹凸、颜色变化，都可以通过 shader 和参数控制。

这类能力适合表现：

- 生长
- 老化
- 腐蚀
- 风吹
- 水流
- 火焰
- 光照变化
- 材质变化

### 3.3 参数化控制

很多视觉效果不是固定动画，而是通过参数控制：

```text
密度
高度
覆盖率
风速
风向
苔藓范围
土壤湿度
模型老化程度
镜头位置
```

这意味着它不仅能做视频，也能做交互工具。用户可以拖动参数，实时看到效果变化。

### 3.4 从技术 demo 到故事化展示

原始技术 demo 通常只证明“能跑”。我们做的改造把它变成：

```text
土壤出现
  -> 车辆进入场景
  -> 苔藓生长
  -> 草逐渐覆盖
  -> 风场变得可见
  -> 完整生态场景成型
```

这说明一个重要方法：技术能力需要被导演化、场景化，才容易被普通观众理解。

## 4. 可扩展应用场景

### 4.1 沉浸式官网

适合做成产品官网首屏或互动页面：

- 汽车品牌：车在荒野、雪地、沙漠、城市中动态展示。
- 户外装备：帐篷、鞋、背包在真实地形中展示。
- 环保项目：荒地变绿地、污染修复、生态恢复。
- 科技产品：用粒子、光场、材质变化展示产品能力。

使用方式：

```text
用户滚动页面
  -> 镜头推进
  -> 场景变化
  -> 产品能力逐段展示
```

### 4.2 产品 3D 展示

可以把“废弃车”换成任意产品模型：

- 手机
- 耳机
- 汽车
- 机器人
- 家具
- 工业设备
- 文创产品

然后用环境和动画讲产品特性，比如：

- 防水：水滴、湿地、雨天。
- 耐用：风沙、腐蚀、撞击。
- 环保：植物生长、自然融合。
- 高性能：光效、粒子、结构拆解。

### 4.3 教学和科普

这个项目很适合讲解：

- Three.js 和 WebGL 的关系。
- Shader 是什么。
- GPU 为什么适合渲染大量草叶。
- 参数如何控制视觉效果。
- 实时渲染和视频渲染的区别。

可以做成课程、短视频、互动课件。

### 4.4 设计提案和概念验证

如果客户或团队不确定某种视觉方向是否可行，可以先用类似方式做快速概念验证：

- 先找模型和贴图。
- 用 Three.js 搭场景。
- 用参数控制核心效果。
- 录制成短片。
- 再决定是否投入正式开发。

这比直接做完整产品更轻、更快。

### 4.5 个人作品和内容发布

对于个人创作者，这类项目可以变成：

- 技术展示视频。
- 作品集案例。
- 教程内容。
- 开源项目讲解。
- 平台短视频内容。

它的传播点不只是“我做了草”，而是：

```text
浏览器里实时跑了一个由 GPU、Shader、Three.js 驱动的 3D 生态场景。
```

## 5. 如何用好这些能力

### 5.1 不要从技术开始，要从场景开始

不要先问：

```text
我能不能加草？
我能不能加 shader？
我能不能加风？
```

更好的问题是：

```text
我要让观众看到什么变化？
我要展示什么产品能力？
这个变化为什么有意义？
```

技术只是表达手段。

### 5.2 把能力组织成故事

一个好的 3D 展示通常有三个阶段：

```text
起点：观众看到一个简单场景
过程：关键能力逐步出现
结果：完整场景形成，观众理解价值
```

当前项目就是：

```text
空地
  -> 车辆
  -> 苔藓
  -> 草
  -> 风
  -> 完整生态
```

### 5.3 用参数控制，而不是写死动画

建议把每种能力都设计成参数：

```text
growthProgress
windStrength
mossCoverage
soilMoisture
cameraProgress
agingAmount
```

这样同一个系统既可以做：

- 视频
- 交互网页
- 演示工具
- 客户定制版本

### 5.4 先做可发布版本，再做复杂系统

如果目标是内容传播，不要无限加功能。更有效的路径是：

```text
一个清楚主题
一个完整场景
一个好封面
一个能听懂的旁白
一个稳定 MP4
```

当前版本已经接近这个状态。
