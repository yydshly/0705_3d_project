# Desktop Companion Storyboard

This storyboard turns the product brief into scroll chapters for a 3D product page.

## One-Sentence Story

Desktop Companion turns AI from a chat box into a persistent, warm presence that lives with the user's desktop, remembers context, reacts emotionally, and grows with the user.

## Chapters

### 1. Presence Appears

Purpose:

```text
Establish that this is not a normal chat app. A companion presence wakes up inside a quiet desktop workspace.
```

Visuals:

- dark desk plane;
- monitor or floating workspace window;
- small companion avatar/light-form;
- soft wake-up glow;
- first greeting bubble.

Scroll-driven 3D state:

- avatar rises from the desktop;
- monitor lights up;
- gentle orbit of small particles.

### 2. Real-Time Conversation

Purpose:

```text
Show the product responding in the moment.
```

Visuals:

- user message card;
- assistant reply card;
- signal lines between user workspace and companion;
- quick expression/mood shift.

Scroll-driven 3D state:

- chat cards slide into place;
- companion turns toward the conversation;
- response pulse appears around the avatar.

### 3. Emotion And Ambient State

Purpose:

```text
Show that the companion has visible state, not only text output.
```

Visuals:

- emotion ring;
- color aura;
- calm/focus/encourage states;
- breathing motion.

Scroll-driven 3D state:

- aura expands;
- material color shifts;
- avatar motion becomes calmer or brighter.

### 4. Long-Term Memory

Purpose:

```text
Explain persistent memory as a visible structure.
```

Visuals:

- memory cards;
- timeline arcs;
- fragments such as goals, preferences, routines, prior conversations;
- cards orbiting and organizing.

Scroll-driven 3D state:

- memory cards assemble around the companion;
- selected memories light up;
- timeline bends into a stable orbit.

### 5. Desktop Interaction

Purpose:

```text
Show the companion relating to the user's real workspace.
```

Visuals:

- windows;
- reminder card;
- file/task panel;
- pointer or small action indicators.

Scroll-driven 3D state:

- desktop windows tilt into 3D layers;
- companion highlights one task;
- reminder panel pins itself near the monitor.

### 6. Customization And Growth

Purpose:

```text
Show that the companion can become personal over time.
```

Visuals:

- appearance swatches;
- personality chips;
- accessory panels;
- final hero workspace.

Scroll-driven 3D state:

- swatches orbit around avatar;
- color/style changes;
- all previous layers settle into one hero composition.

## Film Mode Beats

```text
0-7s: Quiet desktop, companion appears.
7-15s: Conversation cards animate in.
15-23s: Emotion ring and ambient state.
23-33s: Memory fragments assemble.
33-43s: Desktop interaction with windows/tasks.
43-55s: Customization and final hero view.
```

## Caption Draft

```text
1. AI 不必只停留在聊天框里。
2. 它可以成为桌面上的持续陪伴。
3. 对话、情绪、记忆和任务，都能被可视化表达。
4. 这就是一个可以滚动体验，也可以录制成视频的 3D 产品故事页。
```

## Success Criteria

- A viewer understands the product within the first 10 seconds.
- The 3D scene supports the story instead of becoming decoration.
- Each chapter has a clear state change.
- The page can later become a 45-60 second product video.
- The implementation remains isolated from the original template.
